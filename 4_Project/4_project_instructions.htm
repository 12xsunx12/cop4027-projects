Program Description: Network Tic-Tac-Toe!!!

 

We will design a program that enables a single player to play a game of Tic Tac Toe over the Internet with the Tic Tac Toe server. You will use localhost, as described in class, to simplify grading. You will start the server program from a batch file I will provide that will create an instance of the server and one client instance. If you are on MacOS where it is difficult to open consoles programmatically, you can run two programs in Eclipse, or develop on Windows in the Multi-Platform Lab. You must CLEARLY specify how to run your program in the user's manual.


The client will issue the hello message with their player name as a command to join the game. The server will create a game and spawn a thread for the player to play. The server will always make the first move. The server will accept move commands from the client with three parameters: the player number (2), the row of the Tic Tac Toe board (0, 1, or 2) and the column of the board (0, 1, 2).

 

So, our Network Tic-Tac-Toe protocol (N3TP) is:

 

Client Message

Server Response

Meaning

hello <name>

Hello <name> you are player 2

 Player joined the game

move <int><int><int>

game continues <boardData>

    or

<int> WINS!!! <boardData>

 game not over yet

a player has won

 

Notes:

  • The server will assume players make legal moves. The client should check that the move is on the board, but the client DOES NOT have to check if a move is already taken.

  • The goal is to establish the ability to play a game between a human on the client and the server.

  • After each legal move, the board should be drawn on both the client and the server.
